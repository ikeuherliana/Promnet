Git Topic


Git

GitBash adalah adalah software yang biasanya digunakan oleh pengembang untuk mengelola kode sumber, proyek, dan repositori Git di komputer lokal. GitBash menyediakan command-line interface untuk mengoperasikan Git di komputer kita. Sedangkan GitHub adalah sistem kontrol versi yang menggunakan Git dan layanan hosting untuk menyimpan serta mengelola proyek-proyek bagi pengembang software. Para pengembang dapat menyimpan dan berbagi kode sumber mereka ke GitHub untuk berkolaborasi dengan tim termasuk melihat penulis dan tanggal kode dibuat.

Ada beberapa basic Git commands yang sering digunakan untuk mengelola repositori, seperti:
1. git init: untuk meginisialisasi repositori Git baru
2. git status: untuk mengecek status Git atau menampilkan repositori saat ini
3. git log: untuk menampilkan riwayat perubahan commit dalam repositori
4. git add: untuk menambahkan file dan persiapan commit
5. git commit: untuk membuat commit baru dengan file yang sudah ditambahkan sebelumnya
6. git push: untuk mengirim perubahan di repositori lokal ke repositori remote


Branching

Selain itu, ada juga Git branching yang memudahkan pengembang melakukan kolaborasi dan perubahan dalam repositori Git tanpa mengganggu kode yang ada. Perintah-perintah dasarnya sebagai berikut:
1. git branch (nama-branch): untuk membuat cabang baru
2. git branch: untuk menampilkan semua daftar cabang dalam repositori Git
3. git checkout: untuk berpindah dari satu cabang ke cabang yang lain
4. git clone: untuk mengambil clone (salinan) repositori Git dari repositori remote ke repositori lokal
5. git pull: untuk mengambil perubahan terbaru dari repositori remote dan menggabungkannya dengan cabang saat ini
